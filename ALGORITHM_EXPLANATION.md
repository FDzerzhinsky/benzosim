# Детальное объяснение алгоритма пайплайна BenzoSim

## Общее описание

**BenzoSim** — проект для реконструкции характеристики V(H) (объём в зависимости от уровня жидкости) цилиндра, лежащего на боку, на основе эмулированного процесса периодического опустошения ёмкости.

Основная идея: 
1. Генерируем несколько независимых **периодов опустошения** (слива)
2. В каждом периоде снимаем измерения уровня H и объёма V
3. На основе всех этих измерений восстанавливаем единую характеристику V(H)
4. Сравниваем восстановленную характеристику с эталонной (точной геометрией)

---

## Архитектура и классы

### Основные компоненты

1. **`CylinderGeometry`** (`geometry.py`)
   - Точный расчёт объёма V(H) для идеального цилиндра (с учётом помех)
   - Методы: `V_h(h)` (нормированный объём), `dV_h(h)` (производная)
   - Поддержка 4 типов помех: без помех, параллелепипед, цилиндр, множество параллелепипедов

2. **`CylinderExperiment`** (`experiment.py`)
   - Базовый класс эксперимента
   - Инициализация геометрии, добавление шума измерений

3. **`PeriodicDrainingExperiment`** (`periodic_experiment.py`)
   - Наследуется от `CylinderExperiment`
   - Реализует основной пайплайн периодических экспериментов

4. **Утилиты** (`utils.py`)
   - `OriginalSmoother` / `SafeOriginalSmoother` — оригинальный алгоритм сглаживания из Pascal
   - `scipy_smooth()` — сглаживание через SciPy UnivariateSpline
   - Функции для добавления шума, расчёта производных

## Подробный описание пайплайна (9 этапов)

### Этап 1: Инициализация

```
Input: PeriodicExperimentConfig
```

**Конфигурационные параметры:**
- Геометрия цилиндра: R = 100 см (радиус), L = 500 см (длина)
- Количество периодов: n_periods (обычно 5-8)
- Параметры опустошения:
  - `period_start_min/max` — диапазон начальных уровней [см]
  - `period_end_min/max` — диапазон конечных уровней [см]
  - `min_drain_step/max_drain_step` — диапазон шагов опустошения [см]
- Параметры шума:
  - `Del_h` = 0.05 см — амплитуда шума по высоте
  - `Del_V` = 0.0015 — относительная амплитуда шума по объёму
- Сетка интерполяции: `grid_step` = 1.0 см

**Действия:**
1. Инициализируем `CylinderGeometry` с параметрами из конфига
2. Создаём подкаталог для результатов: `plots/periodic_{experiment_name}/`
3. Инициализируем генератор случайных чисел с фиксированным seed

### Этап 2: Генерация периодов опустошения

```
Input: n_periods
Output: List[Dict] periods_data
```

**Для каждого периода:**

1. **Случайный выбор граничных уровней:**
   ```
   start_level ~ Uniform(period_start_min, period_start_max)
   end_level ~ Uniform(period_end_min, period_end_max)
   Гарантия: start_level > end_level (жидкость опускается)
   ```

2. **Генерация точек периода (основной цикл):**

   **Начальная точка (step_index = 0):**
   - `H_ideal_prev = start_level` (точное значение)
   - `V_ideal_prev = R² · L · V_h(start_level/R) / 1000` [в литрах]
   - `H_measured = round(H_ideal_prev, precision)` — округление по заданной точности
   - Шум объёма: `V_measured = V_ideal_prev · (1 + (2x - 1) · Del_V)`, где x ~ U(0,1)

   **Основной цикл (пока H_ideal_prev > end_level):**
   - Случайный шаг опустошения: `drain_step ~ U(min_drain_step, max_drain_step)`
   - Новый уровень: `H_ideal_current = H_ideal_prev - drain_step`
   - Проверка границ: если `H_ideal_current < end_level`, устанавливаем `H_ideal_current = end_level`
   - Точный объём: `V_ideal_current = R² · L · V_h(H_ideal_current/R) / 1000`
   - Округление: `H_measured = round(H_ideal_current, precision)`
   
   **Добавление шума к объёму (ключевой момент):**
   ```
   ideal_increment = V_ideal_current - V_ideal_prev
   x ~ U(0, 1)
   measured_increment = ideal_increment · (1 + (2x - 1) · Del_V)
   V_measured = V_measured_prev + measured_increment  [кумулятивное!]
   ```
   
   - Сохранение точки: (step_index, global_step, H_ideal_prev, H_ideal_current, V_ideal_current, H_measured, V_measured, drain_step)

3. **Результат периода:**
   ```python
   {
       'period_id': int,
       'start_level': float,
       'end_level': float,
       'points': List[Dict]  # Все точки периода
   }
   ```

**Ключевые моменты:**
- Шум добавляется к **приростам** объёма, а не абсолютным значениям
- Шум накапливается (кумулятивный)
- Уровни округляются на каждом шаге
- Используются точные значения V_ideal для расчётов

### Этап 4: Сглаживание данных периодов

```
Input: periods_data
Output: smoothed_data[algorithm] = List[Dict]
```

**Предварительная обработка каждого периода:**

1. Извлечём данные периода:
   ```python
   H_measured = [point['H_measured'] for point in period['points']]
   V_measured = [point['V_measured'] for point in period['points']]
   ```

2. **Развернём массивы** (важно!):
   
   ⚠️ **КЛЮЧЕВОЙ МОМЕНТ АЛГОРИТМА:**
   
   В периодах опустошения уровни **убывают**: H[0] > H[1] > ... > H[n]
   Однако все алгоритмы сглаживания требуют **возрастающие** последовательности!
   
   Поэтому:
   ```python
   H_ascending = H_measured[::-1].copy()  # Разворот в обратном порядке
   V_ascending = V_measured[::-1].copy()
   ```
   
   После сглаживания разворачиваем обратно.

3. **Удаление дубликатов:**
   
   При округлении могут появиться дубликаты по высоте. Их нужно удалить:
   ```python
   for each unique H:
       V = average(V для всех точек с одинаковым H)
   ```
   Получаем: `H_ascending_unique`, `V_ascending_unique`

4. **Применение алгоритма сглаживания:**

   **Алгоритм 1: SciPy (`scipy_smooth`)**
   - Метод: UnivariateSpline
   - Параметр сглаживания: `s = len(data) × 0.1`
   - Эффект: мягкое сглаживание, сохраняет локальные особенности
   
   **Алгоритм 2: Оригинальный Sg_p=1 (`original_sg1`)**
   - Метод: конечные разности + итеративная релаксация
   - Параметры:
     - `I_p = min(5, len(H) // 4)` — буферные точки
     - `Ro = 0.01` — параметр релаксации
     - `max_iter = 15` — максимум итераций
   - Процесс:
     ```
     for iter = 1 to max_iter:
         M[i] = (V[i+1] - V[i]) / h[i] - (V[i] - V[i-1]) / h[i-1]
         V_new[i] = V_old[i] + Ro × M[i]
     ```
   
   **Алгоритм 3: Оригинальный Sg_p=2 (`original_sg2`)**
   - Метод: сплайны с лапласианом
   - Параметры: аналогично Sg_p=1, но `Ro = 0.008`
   - Процесс: вычисляем M-сплайны, затем релаксируем по лапласиану

5. **Очистка NaN/inf:**
   
   Если в результате сглаживания есть нечисловые значения:
   ```python
   V_valid = V[isfinite(V)]
   V_filled = interp1d(H[valid_mask], V[valid_mask])(H)
   ```

6. **Результат сглаживания:**
   ```python
   smoothed_period = {
       'period_id': int,
       'H_measured': np.array,         # Оригинальные измеренные уровни
       'V_measured': np.array,         # Оригинальные измеренные объёмы
       'V_smooth': np.array,           # Сглаженные объёмы (развёрнутые)
       'H_ascending': np.array,        # Уникальные возрастающие уровни
       'V_smooth_ascending': np.array  # Сглаженные объёмы (возрастающие)
   }
   ```

**Результат этапа:**
```python
smoothed_data = {
    'scipy': [period1_smoothed, period2_smoothed, ...],
    'original_sg1': [...],
    'original_sg2': [...]
}
```

### Этап 5: Интерполяция на равномерную сетку

```
Input: smoothed_data, grid_step = 1.0 см
Output: interpolated_data[algorithm] = List[Dict]
```

**⚠️ ОТВЕТ НА ГЛАВНЫЙ ВОПРОС: Как интерполируются границы периодов?**

**Для каждого периода и каждого алгоритма:**

1. **Используем сглаженные возрастающие данные:**
   ```python
   H_ascending = smoothed_period['H_ascending']
   V_smooth_ascending = smoothed_period['V_smooth_ascending']
   ```

2. **Создаём равномерную сетку:**
   ```python
   H_min = min(H_ascending)
   H_max = max(H_ascending)
   
   # Расширяем сетку на один узел за границы (для полноты)
   H_min_ext = max(0.0, H_min - grid_step)
   H_max_ext = min(2×R, H_max + grid_step)
   
   # Создаём сетку с шагом grid_step
   H_grid = [H_min_ext, H_min_ext + grid_step, ..., H_max_ext]
   ```

3. **Линейная интерполяция на сетку:**
   
   **КЛЮЧЕВОЙ АЛГОРИТМ ИНТЕРПОЛЯЦИИ:**
   
   ```python
   # Фильтруем только конечные (валидные) значения
   valid_mask = isfinite(V_smooth_ascending)
   H_valid = H_ascending[valid_mask]
   V_valid = V_smooth_ascending[valid_mask]
   
   # Сортируем по H (должны быть в порядке возрастания)
   sort_idx = argsort(H_valid)
   H_valid = H_valid[sort_idx]
   V_valid = V_valid[sort_idx]
   
   # Интерполируем на сетку ВНУТРИ диапазона [H_valid[0], H_valid[-1]]
   V_grid = interp(H_grid, H_valid, V_valid)
   
   # НА КРАЯХ (вне диапазона валидных точек) устанавливаем NaN
   V_grid[H_grid < H_valid[0]] = NaN
   V_grid[H_grid > H_valid[-1]] = NaN
   ```
   
   **Почему так?**
   - Интерполяция внутри диапазона — стандартная линейная интерполяция
   - Экстраполяция на краях опасна (может дать нереальные значения)
   - NaN на краях указывает на отсутствие информации вне диапазона периода

4. **Разворот обратно:**
   ```python
   H_grid = H_grid_ascending[::-1]
   V_grid = V_grid_ascending[::-1]
   ```
   
   Делаем для согласованности с оригинальным убывающим порядком данных.

5. **Результат интерполяции:**
   ```python
   interpolated_period = {
       'period_id': int,
       'H_grid': np.array,            # Сетка высот (убывающая, для согласованности)
       'V_grid': np.array,            # Интерполированные объёмы на сетке
       'H_original': np.array,        # Оригинальные уровни
       'V_smooth_original': np.array  # Оригинальные сглаженные объёмы
   }
   ```

**Результат этапа:**
```python
interpolated_data = {
    'scipy': [period1_interp, period2_interp, ...],
    'original_sg1': [...],
    'original_sg2': [...]
}
```

**Итого: каждый период теперь представлен на единой сетке высот!**

### Этап 6: Создание объединённой характеристики

```
Input: interpolated_data, grid_step = 1.0 см
Output: combined_characteristics[algorithm] = {H, V, V_raw, V_stronger}
```

**Это ключевой этап — мы объединяем данные из ВСЕХ периодов в ОДНУ характеристику!**

**Алгоритм агрегации:**

1. **Создаём глобальную сетку:**
   ```python
   H_global = arange(0, 2×R + grid_step, grid_step)
   # Для R=100: H_global = [0, 1, 2, ..., 199, 200]
   ```

2. **Накапливаем объёмы для каждого узла сетки:**
   ```python
   height_volumes = {}  # ключ: высота (см), значение: список (V, period_id)
   
   for each algorithm:
       for each interpolated_period:
           for (h, v) in zip(H_grid, V_grid):
               if isfinite(v):
                   h_index = round(h / grid_step)
                   h_key = float(h_index × grid_step)  # Привязка к сетке
                   
                   # Избегаем дублей от одного периода
                   if not any(rec.period_id == current_period_id for rec in height_volumes[h_key]):
                       height_volumes[h_key].append((v, period_id))
   ```
   
   **Результат:**
   - Для каждой высоты h имеем список объёмов из разных периодов
   - Пример: `height_volumes[100.0] = [(15350, period_1), (15380, period_2), (15340, period_5)]`

3. **Усреднение по периодам:**
   ```python
   H_combined = []
   V_combined = []
   
   for h_key in sorted(height_volumes.keys()):
       volumes = height_volumes[h_key]
       V_avg = mean([v for (v, _) in volumes])
       
       H_combined.append(h_key)
       V_combined.append(V_avg)
   ```
   
   **Результат:**
   - Единая характеристика V(H) на основе всех периодов
   - На каждой высоте — среднее значение объёма по всем периодам, прошедшим через эту высоту

4. **Сглаживание объединённой характеристики:**
   
   Объединённая характеристика часто шумная (усреднение не полностью убирает шум).
   Поэтому применяем второе сглаживание SciPy:
   ```python
   V_smoothed = scipy_smooth(H_combined, V_combined, 
                            I_beg=0, I_end=len(H_combined)-1, 
                            smooth_factor=0.1)
   ```

5. **Усиленное сглаживание (опционально):**
   
   Для сравнения создаём более сильно сглаженную версию:
   ```python
   V_smoothed_stronger = scipy_smooth(H_combined, V_combined,
                                     smooth_factor=100.0)  # На два порядка больше
   ```

6. **Сохранение результатов:**
   ```python
   combined_characteristics[algorithm] = {
       'H': H_combined,           # Уровни
       'V': V_smoothed,          # Сглаженные объёмы (основная версия)
       'V_raw': V_combined,      # Объёмы до второго сглаживания
       'V_stronger': V_smoothed_stronger,  # Усиленно сглаженные
       'periods_count': len(height_volumes)  # Количество уникальных узлов
   }
   ```

7. **Сохранение в файлы:**
   - `cloud_before_agg_{algorithm}.csv` — все точки ДО агрегации
   - `combined_{algorithm}.csv` — объединённая характеристика (сглаженная)
   - `counts_{algorithm}.csv` — количество периодов в каждом узле
   - `combined_{algorithm}_strong.csv` — все три версии сглаживания для сравнения

**Ключевые идеи этого этапа:**
1. **Облако точек** — для каждой высоты накапливаются значения из разных периодов
2. **Усреднение** — убирает индивидуальный шум каждого периода
3. **Второе сглаживание** — убирает оставшийся шум от усреднения
4. **Все три версии сохраняются** для анализа эффекта сглаживания

### Этап 7: Сравнение с эталонной характеристикой

```
Input: combined_characteristics, geometry (для расчёта точной V(H))
Output: Графики сравнения
```

**Для каждого алгоритма:**

1. Вычисляем точные значения V для всех H в combined_characteristics:
   ```python
   V_exact = [R² × L × V_h(h/R) / 1000 for h in H_combined]
   ```

2. Строим график:
   ```
   - синяя кривая: точная V(H) из геометрии
   - красная кривая: восстановленная V(H) из периодов
   - (опционально) серая кривая: облако отдельных периодов
   ```

3. Вычисляем метрики ошибки:
   ```
   error = |V_recovered - V_exact|
   relative_error = error / V_exact
   ```

4. Сохраняем график: `combined_vs_geometry_{algorithm}.png`

**Для каждой характеристики:**

   ```python
   dV_dH = gradient(V, H)  # Конечные разности
   ```

   ```python
   dV_dH_exact = [R² × L × dV_h(h/R) / 1000 for h in H]
   ```

3. Строим графики сравнения:
   ```
   - кривая производной восстановленной V(H)
   - кривая точной производной
   - анализ отклонений
   ```

4. Сохраняем: `derivative_analysis_{algorithm}_combined.png`

## Критические интерполяционные моменты

### 1. Развёртывание массивов (Этап 4)

```python
# В периодах опустошения уровни убывают!
H_measured = [200, 195, 190, 180, 150, ...]  # убывает
V_measured = [20000, 19500, 19000, 18000, 15000, ...]  # может колебаться из-за шума

# Разворачиваем для сглаживания
H_ascending = [150, 180, 190, 195, 200]  # возрастает!
V_ascending = [15000, 18000, 19000, 19500, 20000]  # в соответствующем порядке
```

### 2. Удаление дубликатов (Этап 4)

```python
# После округления может быть:
H_unique = [150.0, 180.0, 180.0, 190.0, 195.0]
V = [15000, 18100, 17900, 19000, 19500]

# После удаления дубликатов:
H_unique = [150.0, 180.0, 190.0, 195.0]
V_unique = [15000, 18000, 19000, 19500]  # 18100 и 17900 усредняются
```

### 3. Интерполяция на сетку (Этап 5)

```python
# Период покрыл уровни [180, 200]
H_ascending_unique = [180, 185, 190, 195, 200]
V_smooth_ascending = [18000, 18500, 19000, 19500, 20000]

# Сетка (1 см):
H_grid = [0, 1, 2, ..., 179, 180, 181, ..., 200, 201, ..., 200]

# После интерполяции:
V_grid = [NaN, NaN, ..., NaN, 18000, 18250, 18500, ..., 20000, NaN, NaN, ...]
           ↑ вне диапазона ↑        ↑ интерполяция внутри ↑       ↑ вне диапазона

# Ключ: NaN на краях означают "нет данных здесь"
```

## Ключевые выводы

### Что делает алгоритм интерполяции особенным?

1. **Развёртывание массивов** — необходимо для работы алгоритмов, требующих монотонности
2. **Линейная интерполяция внутри диапазона** — надёжный метод
3. **NaN на краях** — честно показывает, где нет данных
4. **Привязка к сетке через индексы** — избегает ошибок округления с плавающей точкой
5. **Агрегация по всем периодам** — усреднение уменьшает влияние отдельных шумов

### Как работает восстановление характеристики?

- **Облако точек** собирает все значения V для каждой высоты H
- **Усреднение** в каждой точке сетки убирает индивидуальный шум
- **Второе сглаживание** объединённой характеристики убирает оставшийся шум
- **Результат** — гладкая монотонная кривая V(H), очень близкая к точной геометрии

### Почему три алгоритма сглаживания?

1. **SciPy** — быстрый, универсальный, надёжный
2. **Original Sg_p=1** — физически мотивирован конечными разностями
3. **Original Sg_p=2** — более гладкий результат через сплайны

Их сравнение показывает чувствительность результата к выбору алгоритма.
