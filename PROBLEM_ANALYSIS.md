  Нужно:

    period_end_min/max = 0-50 см       ← нижний край
  
  ⇒ Несколько периодов начнут с 150-200 см (покроют верх)
  ⇒ Несколько периодов закончат с 0-50 см (покроют низ)
  
  ПЛЮСЫ:
    - Просто меняем конфиг
    - Пайплайн не меняется
  
  МИНУСЫ:
    - Может потеряться перекрытие в середине [50, 150]

ВАРИАНТ 2: ГАРАНТИРОВАТЬ периоды, покрывающие края (ПРАВИЛЬНО)

  Выбрать часть периодов специально для края:
    - 2-3 периода стартуют высоко (160-200 см) и идут низко
    - 2-3 периода стартуют низко (0-100 см) и заканчиваются на нуле
    - Остальные периоды случайные для заполнения середины
  
  ПЛЮСЫ:
    - Гарантированное полное покрытие
    - Хорошее перекрытие везде
  
  МИНУСЫ:
    - Требует изменения логики в generate_periods()

ВАРИАНТ 3: Экстраполяция на краях вместо NaN (РИСКОВАННО)

  В Шаге 4 (интерполяция), вместо:
    V_grid[H_grid < H_valid[0]] = NaN
    V_grid[H_grid > H_valid[-1]] = NaN
  
  Использовать линейную экстраполяцию:
    slope_left = (V_valid[1] - V_valid[0]) / (H_valid[1] - H_valid[0])
    V_grid[H < H_valid[0]] = V_valid[0] + slope_left × (H_grid - H_valid[0])
  
  ПЛЮСЫ:
    - Заполнит "дыры" в финальной характеристике
  
  МИНУСЫ:
    - Экстраполяция опасна (может дать нереальные значения)
    - Нарушает принцип "честной интерполяции"
    - Может скрыть настоящую проблему с конфигом
