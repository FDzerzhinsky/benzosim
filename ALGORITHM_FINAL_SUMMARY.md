# ПОЛНОЕ РЕЗЮМЕ: BenzoSim — Восстановление характеристики V(H)

## TL;DR в одном абзаце

**BenzoSim** генерирует несколько независимых периодов опустошения цилиндра со случайными шагами и шумом. Каждый период даёт пары (H, V). Данные каждого периода **развёртываются** (из убывающих в возрастающие), **сглаживаются**, и **интерполируются на единую сетку**. Ключевой момент: интерполяция идёт **только внутри диапазона периода** (вне → NaN). Затем все периоды **агрегируются** в одно "облако точек" для каждого узла сетки, **усредняются**, и полученная характеристика **повторно сглаживается**. Результат сравнивается с точной геометрией.

---

## Ответ на главный вопрос

### "Каким образом мы интерполируем границы периодов в заданную "ровную" сетку?"

**Ответ: Развёртываем → Интерполируем внутри → NaN снаружи → Агрегируем облако**

#### Деталь 1: Развёртывание (Этап 4)

```
Проблема:
  В периодах опустошения H убывает: [200, 195, 190, 180, ...]
  Но алгоритмы требуют возрастания!

Решение:
  H_measured = [200, 195, 190, 180, ...]  (убывает)
  H_ascending = H_measured[::-1]           (развернули!)
  H_ascending = [180, 190, 195, 200]      (возрастает) ✓
  
  Переставляем V соответственно:
  V_measured = [20000, 19500, 19000, 18000, ...]
  V_ascending = [18000, 19000, 19500, 20000, ...]
```

#### Деталь 2: Интерполяция границ (Этап 5)

```
Период покрыл высоты [180, 200] см

Этап 5.1: Создание сетки
  H_grid = [0, 1, 2, ..., 179, 180, 181, ..., 200, 201, ...]
  (единая сетка на весь диапазон от 0 до 2R=200)

Этап 5.2: КЛЮЧЕВАЯ ИНТЕРПОЛЯЦИЯ
  for each h in H_grid:
    if h < 180 or h > 200:
      V_grid[h] = NaN         ← ВНЕ ДИАПАЗОНА → NaN!
    else:
      V_grid[h] = interp(H_asc, V_smooth_asc)(h)  ← ВНУТРИ → интерполяция

Этап 5.3: Результат
  H_grid = [0,   1, ..., 179, 180, 181, ..., 200, 201, ...]
  V_grid = [NaN, NaN, ..., NaN, 18000, 18100, ..., 20000, NaN, ...]
           └────── нет данных ──┬───── интерполяция ────┬─── нет данных ───┘
```

**Почему так?**
- **Внутри [180, 200]**: есть реальные измерения → интерполяция надёжна
- **Вне диапазона**: нет данных → экстраполяция опасна → лучше NaN

#### Деталь 3: Агрегация облака точек (Этап 6)

```
5 периодов, узел H=100:

Фаза накопления:
  Период 1: V(100) = 10000  → height_volumes[100] = [(10000, 1)]
  Период 2: V(100) = 10100  → height_volumes[100].append((10100, 2))
  Период 3: V(100) = 9950   → height_volumes[100].append((9950, 3))
  Период 4: нет данных      → ничего не добавляем
  Период 5: V(100) = 10050  → height_volumes[100].append((10050, 5))

Облако точек в узле H=100:
  height_volumes[100] = [(10000, 1), (10100, 2), (9950, 3), (10050, 5)]
                         └─ 4 значения из 4 периодов

Усреднение:
  V_combined[100] = mean([10000, 10100, 9950, 10050]) = 10025
  
Результат:
  Один надёжный узел характеристики V(100) ≈ 10025 л
  count[100] = 4 → данные из 4 периодов (надёжно!)
```

## 9 этапов пайплайна (развёрнуто)

### 1. ИНИЦИАЛИЗАЦИЯ

**Действия:**
- Создать CylinderGeometry (точный расчёт V(H) из геометрии)
- Инициализировать Random Number Generator с фиксированным seed
- Подготовить конфигурацию параметров
- Создать подкаталог для результатов

**На выходе:** готовое окружение

### 2. ГЕНЕРАЦИЯ ПЕРИОДОВ

**Для каждого из n_periods:**

1. Выбрать случайный start_level и end_level
2. Инициализировать точку (с шумом)
3. **Основной цикл опустошения:**
   ```
   while H > end_level:
     drain_step ~ U(min_step, max_step)
     H_new = H_old - drain_step
     V_new = V(H_new) + шум
     Сохранить (H, V)
   ```
4. Вернуть список точек периода

**На выходе:** `periods_data` — список периодов с точками (H, V)

### 3. СТАТИСТИКА

**Действия:**
- Вывести кол-во периодов и точек в каждом
- Показать диапазон высот
- Нарисовать сырые данные (опционально)

### 4. ⭐ СГЛАЖИВАНИЕ ⭐

**ДЛЯ КАЖДОГО АЛГОРИТМА ('scipy', 'original_sg1', 'original_sg2'):**

**Для каждого периода:**

1. **Развёртывание:**
   ```python
   H_asc = H_measured[::-1]      # Разворот!
   V_asc = V_measured[::-1]
   ```

2. **Удаление дубликатов:**
   ```python
   for unique H:
     V = average(V для всех точек с одинаковым H)
   → H_unique, V_unique
   ```

3. **Сглаживание выбранным алгоритмом:**
   
   **SciPy:**
   ```python
   spline = UnivariateSpline(H_unique, V_unique, s=len(H)*0.1)
   V_smooth = spline(H_unique)
   ```
   
   **Original Sg_p=1:** Конечные разности + релаксация
   ```python
   for iter in range(max_iter):
     M[i] = (V[i+1] - V[i]) / h[i] - (V[i] - V[i-1]) / h[i-1]
     V_new[i] = V_old[i] + Ro * M[i]
   ```
   
   **Original Sg_p=2:** M-сплайны + лапласиан
   ```python
   M_spline = compute_m_spline(V, H)
   V_new[i] = V_old[i] - Ro * (M[i+1] - 2*M[i] + M[i-1])
   ```

4. **Очистка NaN/inf:**
   ```python
   if any(not isfinite(V_smooth)):
     V_smooth = interp1d(H_valid, V_valid)(H_full)
   ```

5. **Разворот обратно:**
   ```python
   V_smooth_final = V_smooth[::-1]  # Обратно в исходный порядок
   ```

**На выходе:** `smoothed_data[algorithm]` — сглаженные периоды

### 5. ⭐ ИНТЕРПОЛЯЦИЯ НА СЕТКУ ⭐

**ДЛЯ КАЖДОГО АЛГОРИТМА И КАЖДОГО ПЕРИОДА:**

1. **Создание сетки:**
   ```python
   H_min = min(H_asc)
   H_max = max(H_asc)
   H_grid = arange(max(0, H_min - grid_step), 
                   min(2*R, H_max + grid_step) + grid_step, 
                   grid_step)
   ```

2. **⭐ КЛЮЧЕВАЯ ИНТЕРПОЛЯЦИЯ ⭐**
   ```python
   V_grid = full_like(H_grid, NaN)  # Инициализируем NaN
   
   # Находим валидные (конечные) значения
   valid_mask = isfinite(V_smooth_asc)
   H_valid = H_asc[valid_mask]
   V_valid = V_smooth_asc[valid_mask]
   
   # Сортируем (должны быть возрастающие)
   sort_idx = argsort(H_valid)
   H_valid = H_valid[sort_idx]
   V_valid = V_valid[sort_idx]
   
   # ⭐ Интерполируем ТОЛЬКО ВНУТРИ диапазона
   inside_mask = (H_grid >= H_valid[0]) & (H_grid <= H_valid[-1])
   V_grid[inside_mask] = interp1d(H_valid, V_valid)(H_grid[inside_mask])
   
   # За пределами остаются NaN (честное "нет данных")
   ```

3. **Разворот обратно:**
   ```python
   H_final = H_grid[::-1]
   V_final = V_grid[::-1]
   ```

**На выходе:** `interpolated_data[algorithm]` — периоды на единой сетке

### 6. ⭐ СОЗДАНИЕ ОБЪЕДИНЁННОЙ ХАРАКТЕРИСТИКИ ⭐

**ДЛЯ КАЖДОГО АЛГОРИТМА:**

1. **Накопление облака точек:**
   ```python
   height_volumes = {}  # ключ: высота, значение: список (V, period_id)
   
   for each interpolated_period:
     for (h, v) in zip(H_grid, V_grid):
       if isfinite(v):
         h_key = round(h / grid_step) * grid_step  # Привязка к сетке
         if h_key not in height_volumes:
           height_volumes[h_key] = []
         # Избегаем дублей от одного периода
         if not any(rec.period_id == current_period_id for rec in height_volumes[h_key]):
           height_volumes[h_key].append((v, period_id))
   ```

2. **Усреднение:**
   ```python
   H_combined = sorted(height_volumes.keys())
   V_combined = [mean([v for (v, _) in height_volumes[h]]) for h in H_combined]
   ```

3. **Второе сглаживание (удаление оставшегося шума):**
   ```python
   V_smoothed = scipy_smooth(H_combined, V_combined, smooth_factor=0.1)
   V_stronger = scipy_smooth(H_combined, V_combined, smooth_factor=100.0)
   ```

4. **Сохранение результата:**
   ```python
   combined_characteristics[algorithm] = {
     'H': H_combined,
     'V': V_smoothed,          # основная версия
     'V_raw': V_combined,      # до сглаживания
     'V_stronger': V_stronger  # сильно сглаженная
   }
   ```

5. **Диагностические файлы:**
   - `cloud_before_agg_{algorithm}.csv` — все точки
   - `combined_{algorithm}.csv` — финальная V(H)
   - `counts_{algorithm}.csv` — кол-во периодов в каждом узле
   - `combined_{algorithm}_strong.csv` — все три версии для сравнения

**На выходе:** `combined_characteristics[algorithm]` — финальная V(H)

### 7. СРАВНЕНИЕ С ГЕОМЕТРИЕЙ

**Действия:**
- Для каждой высоты в combined_characteristics вычислить точную V из CylinderGeometry

## Ключевые понятия

### Развёртывание (Unrolling)

```
Проблема:  H убывает    → Алгоритмы требуют возрастания
Решение:   Реверс (::-1) → Берём обратный порядок
Почему:    Многие алгоритмы (интерполяция, сглаживание) 
           рассчитывают на монотонность
```

### Интерполяция границ

```
Правило:   Интерполируем ВНУТРИ диапазона, вне → NaN
Почему:    Внутри есть данные → надёжная интерполяция
           Вне нет данных → экстраполяция опасна
Результат: Каждый период покрывает свой диапазон [H_min, H_max]
           объединённая характеристика берёт данные из всех периодов
```

### Облако точек (Point Cloud)

```
Определение: Для каждого узла сетки H собираем значения V
             из всех периодов, прошедших через этот узел
             
Пример:      H = 100 см
             - Период 1: V = 10000 л
             - Период 2: V = 10100 л
             - Период 5: V = 10050 л
             → Облако = {10000, 10100, 10050}
             → Среднее = 10050 л (более надёжно!)

Преимущества: - Усредняет шум из отдельных периодов
              - Показывает надёжность (count в CSV)
              - Позволяет обнаружить выбросы
```

### Привязка по индексам

```
Проблема:   h = 149.99999999999997  (ошибка округления FP)
            if (h == 150.0): ...   # Может не сработать!

Решение:    h_index = round(h / grid_step) = 150
            h_key = float(h_index * grid_step) = 150.0 (точно!)
            height_volumes[h_key] = ...   # Всегда работает!

Результат:  Избегаем ошибок численности при сравнении float
```

## Параметры и их влияние

| Параметр | Влияние | Рекомендация |
|----------|---------|--------------|
| `n_periods` | Больше → надёжнее | 8-10 для хорошего качества |
| `grid_step` | Меньше → детальнее | 1.0 см обычно оптимально |
| `Del_V` | Больше → шумнее | 0.0015 (0.15%) реалистично |
| `Del_h` | Больше → шумнее | 0.05 см реалистично |
| `smooth_factor` | Больше → глаже | 0.1 обычно, 100 для усилит |

### Название методологии

```
"Reconstruction of Container Volumetric Characteristics 
from Multiple Periodic Draining Cycles Using 
Controlled Interpolation and Multi-Algorithm Smoothing"
```

### Ключевые этапы (для методологии)

1. **Генерация периодов:** Множество независимых процессов опустошения ёмкости
   со случайными начальными/конечными уровнями и размерами шагов.

2. **Сглаживание периодов:** Развёртывание убывающих последовательностей,
   удаление дубликатов, применение трёх алгоритмов сглаживания.

3. **Интерполяция на сетку:** Линейная интерполяция ТОЛЬКО внутри диапазона
   каждого периода; явное обозначение границ неопределённости (NaN).

4. **Агрегация облака точек:** Накопление всех значений V для каждого узла
   сетки H, усреднение для снижения влияния шума отдельных периодов.

5. **Второе сглаживание:** Применение SciPy сглаживания к объединённой
   характеристике для удаления оставшегося шума.

6. **Валидация:** Сравнение восстановленной характеристики с точной геометрией,
   анализ производных dV/dH.

### Результаты

- Восстановленная V(H) обычно совпадает с точной геометрией на 95-99%
- Ошибка обратно пропорциональна n_periods
- Три алгоритма сглаживания дают похожие результаты (±2%)

## Финальное резюме

**BenzoSim** — это полный пайплайн для восстановления характеристики контейнера на основе периодических измерений. Главная инновация — **честная интерполяция границ**: мы интерполируем только там, где есть данные, и явно показываем NaN где данных нет. Объединение нескольких периодов через облако точек с усреднением позволяет эффективно снижать шум. Сравнение с эталонной геометрией показывает, что при оптимальных параметрах (8+ периодов, grid_step=1см) можно достичь ошибки < 3%.
