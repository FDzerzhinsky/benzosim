# Визуальная схема пайплайна BenzoSim

## Общая архитектура (High Level)

```
┌─────────────────────────────────────────────────────────────────┐
│                    PeriodicDrainingExperiment                    │
│                                                                   │
│  Генерируем несколько независимых периодов опустошения ёмкости  │
│  Восстанавливаем характеристику V(H) из этих периодов           │
│  Сравниваем с эталонной геометрией                              │
└─────────────────────────────────────────────────────────────────┘
```

## Полный пайплайн (с основными этапами)

```

│ ЭТАП 1: ИНИЦИАЛИЗАЦИЯ                                            │

               │
               ▼

│ ЭТАП 2: ГЕНЕРАЦИЯ ПЕРИОДОВ (n_periods = 5-8)                   │
│                                                                   │
│ Для каждого периода:                                             │
│ ┌────────────────────────────────────────────────────────────┐  │
│ │ start_level = Random(80-180см)                             │  │
│ │ end_level = Random(20-60см)                                │  │
│ │                                                             │  │
│ │ Цикл опустошения:                                           │  │
│ │   while current_H > end_level:                              │  │

│ │     Сохранить точку (H, V)                                 │  │
│ └────────────────────────────────────────────────────────────┘  │
│                                                                   │
│ ВЫХОД: periods_data                                              │
│   [                                                               │
│     {period_id: 0, points: [(H0,V0), (H1,V1), ...]},           │
│     {period_id: 1, points: [(H0,V0), (H1,V1), ...]},           │
│     ...                                                           │
│   ]                                                               │
└──────────────┬──────────────────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────────────────────┐
│ ЭТАП 3: СТАТИСТИКА И ВИЗУАЛИЗАЦИЯ                               │
│ - Выводим информацию о периодах                                  │
│ - Строим графики сырых данных                                    │
└──────────────┬──────────────────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────────────────────┐
│ ЭТАП 4: СГЛАЖИВАНИЕ ДАННЫХ ПЕРИОДОВ                             │
│                                                                   │
│ Для каждого алгоритма ('scipy', 'original_sg1', 'original_sg2'): │
│ ┌────────────────────────────────────────────────────────────┐  │
│ │ Для каждого периода:                                       │  │
│ │ ┌──────────────────────────────────────────────────────┐   │  │
│ │ │ 1. Извлечём H_measured, V_measured                 │   │  │

│ └────────────────────────────────────────────────────────┘  │
│                                                                   │
│ ВЫХОД: smoothed_data                                             │
```

## Деталь: Интерполяция границ периодов (Этап 5)

### Визуальное представление интерполяции

```
Период 1: опустошение с 200 см до 150 см

Исходные данные (убывающие):
  H_meas:     200  195  190  180  170  160  150
  V_meas:   20000 19500 19200 18500 17500 16500 15000

После развёртывания (возрастающие):
  H_asc:      150  160  170  180  190  195  200
  V_asc:     15000 16500 17500 18500 19200 19500 20000

После сглаживания:
  H_asc_smooth: 150  160  170  180  190  195  200
  V_smooth:    15050 16480 17480 18480 19180 19480 19950

Равномерная сетка (шаг 1 см):
  0    1    2   ...  149  150  151  152  ...  200  201  ...
  |    |    |        |    |    |    |         |    |
  NaN  NaN  NaN ... NaN [150→200 интерполяция] NaN  NaN ...

Интерполяция на сетку:
  ┌────── Вне диапазона (NaN)
  │        ┌─────────── Интерполированные значения
  │        │                           ┌──── Вне диапазона (NaN)
  ▼        ▼                           ▼
  NaN ... NaN  15050  15150  15250 ... 19950  NaN  NaN ...
             ▲                          ▲
         150 см                      200 см

Результат:
  V_grid имеет:
  - NaN для H < 150 см (нет данных из этого периода)
  - Интерполированные значения для 150 ≤ H ≤ 200
  - NaN для H > 200 см (нет данных из этого периода)
```

### Почему именно так интерполируются границы?

**Интерполяция (не экстраполяция!) — это главное:**

```
Внутри диапазона [150, 200]:
  ✓ Значения основаны на реальных измерениях
  ✓ Линейная интерполяция надёжна и консервативна
  ✓ Даёт физически правдоподобные результаты

За пределами [150, 200]:
  ✗ Нет реальных измерений в этом диапазоне
  ✗ Экстраполяция может дать нереальные значения
  ✗ Лучше явно показать "нет данных" через NaN

Агрегация в ЭТАП 6:
  - Узлы с достаточным кол-вом периодов (count > 1) → надёжные значения
  - Узлы с одним периодом → менее надёжны
  - CSV файл counts показывает эту информацию
```

### Привязка к сетке через индексы

```python
# Проблема: ошибки округления в плавающей точке
h_raw = 149.99999999999997  # Должно быть 150.0!
if abs(h_raw - 150.0) < 1e-6:  # Опасно!
    # Может не сработать из-за численных ошибок

# Решение: привязываем через индексы

```

## Деталь: Агрегация облака точек (Этап 6)

### Визуализация "облака"

```
5 периодов прошли через высоту H = 100 см:

Период 1: V = 12000 л
Период 2: V = 12050 л
Период 3: V = 11950 л  ← Выбросы (шум)
Период 4: V = 12100 л  ← из-за шумов
Период 5: V = 12000 л

          │       *  Период 4
          │   *      Период 3
          │   *      Период 1, 5
          │   *      Период 2
          │   ▼▼▼▼▼
       V  │  V_avg = (12000+12050+11950+12100+12000)/5 = 12020
          │   ╳────── Среднее значение (более надёжно!)
          │
         12000 ├──────────────────────────►
              H = 100 см
```

### Дисперсия по периодам — это хорошо!

```
Если V в разных периодах для одной H сильно разнятся:
  → Может быть из-за шума в измерениях
  → Или из-за реальной вариативности
  → Усреднение автоматически снижает влияние выбросов

CSV файл counts_{algorithm}.csv показывает:
  - Узлы с count = 5 → V_avg из 5 периодов (надёжно!)
  - Узлы с count = 1 → V_avg = V (менее надёжно)
  - Узлы с count = 0 → нет данных (пусто)
```

## Алгоритм в виде псевдокода

```python
def run_periodic_experiment():
    # Этап 1: Инициализация
    geometry = CylinderGeometry(config)
    
    # Этап 2: Генерация периодов
    periods = []
    for period_id in range(n_periods):
        period = generate_single_period(geometry, config)
        periods.append(period)
    
    # Этап 4: Сглаживание
    smoothed = {}
    for algorithm in ['scipy', 'original_sg1', 'original_sg2']:
        smoothed[algorithm] = []
        for period in periods:
            # Извлечём H_meas, V_meas (убывающие)
            H_asc = H_meas[::-1]  # ⭐ Развернём!
            V_asc = V_meas[::-1]
            
            # Удалим дубликаты
            H_unique, V_unique = remove_duplicates(H_asc, V_asc)
            
            # Сгладим
            V_smooth = smooth_with_algorithm(algorithm, H_unique, V_unique)
            
            # Развернём обратно
            smoothed[algorithm].append({
                'H_asc': H_unique,
                'V_smooth_asc': V_smooth
            })
    
    # Этап 5: Интерполяция
    interpolated = {}
    for algorithm in smoothing_algorithms:
        interpolated[algorithm] = []
        for period_smooth in smoothed[algorithm]:
            H_asc = period_smooth['H_asc']
            V_asc = period_smooth['V_smooth_asc']
            
            # Создаём сетку
            H_grid = arange(0, 2*R + grid_step, grid_step)
            
            # ⭐ Интерполируем ТОЛЬКО внутри диапазона
            V_grid = np.full_like(H_grid, np.nan)
            mask = (H_grid >= H_asc[0]) & (H_grid <= H_asc[-1])
            V_grid[mask] = interp1d(H_asc, V_asc)(H_grid[mask])
            
            interpolated[algorithm].append({
                'H_grid': H_grid,
                'V_grid': V_grid
            })
    
    # Этап 6: Агрегация
    combined = {}
    for algorithm in smoothing_algorithms:
        # Накопляем облако точек
        height_volumes = {}
        for period_interp in interpolated[algorithm]:
            for h, v in zip(period_interp['H_grid'], period_interp['V_grid']):
                if not isnan(v):
                    h_key = round(h / grid_step) * grid_step  # Привязка
                    if h_key not in height_volumes:
                        height_volumes[h_key] = []
                    height_volumes[h_key].append(v)
        
        # Усреднение
        H_combined = sorted(height_volumes.keys())
        V_combined = [mean(height_volumes[h]) for h in H_combined]
        
        # Второе сглаживание
        V_smoothed = scipy_smooth(H_combined, V_combined)
        
        combined[algorithm] = {'H': H_combined, 'V': V_smoothed}
    
    # Этапы 7-9: Визуализация и сравнение
    for algorithm in smoothing_algorithms:
        # Вычисляем точную геометрию
        V_exact = [geometry.V_h(h/R) * R*R*L for h in combined[algorithm]['H']]
        
        # Сравниваем и визуализируем
        plot_comparison(combined[algorithm], V_exact)
```

## Ключевые особенности интерполяции

| Особенность | Почему | Как |
|------------|--------|-----|
| **Развёртывание** | Алгоритмы требуют возрастания | `H_asc = H_meas[::-1]` |
| **Удаление дубликатов** | Округление создаёт дубликаты | Усреднение V для одинаковых H |
| **Интерполяция внутри** | Надёжная, основана на данных | `interp1d(H_valid, V_valid)` |
| **NaN на краях** | Честно показываем отсутствие данных | `V[out_of_range] = NaN` |
| **Привязка по индексам** | Избегаем ошибок FP округления | `h_key = round(h/step) * step` |
| **Агрегация по сетке** | Объединяем периоды | `for h: V[h] = mean(all_V[h])` |
| **Второе сглаживание** | Убираем оставшийся шум | `scipy_smooth(H, V_avg)` |
