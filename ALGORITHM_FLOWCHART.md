## Справочная таблица: Что происходит в каждом узле сетки

```
ПРИМЕР: 5 периодов, анализируем узлы H ∈ [50, 200]

H=50см:
  - Период 1: V=5000 л
  - Периоды 2-5: нет данных
  → counts[50] = 1 ← менее надёжно
  → V_avg[50] = 5000

H=100см:
  - Периоды 1,3,4: V=10000, 10050, 9950 л (облако из 3)
  - Периоды 2,5: нет данных
  → counts[100] = 3 ← хорошо!
  → V_avg[100] = (10000 + 10050 + 9950) / 3 = 10000

H=150см:
  - ВСЕ 5 периодов имеют данные!
  - Облако из 5 значений
  → counts[150] = 5 ← очень надёжно!
  → V_avg[150] = mean([...]) ← минимум шума!

H=200см:
  - Только Период 1
  - Нет данных из других
  → counts[200] = 1 ← менее надёжно
  → V_avg[200] = одно значение

ЗАКОНОМЕРНОСТЬ:
  counts[H] → количество периодов, прошедших через H
  
  count = 1   → данные из одного периода → высокий шум
  count = 2-3 → данные из нескольких   → средний шум
  count = 4-5 → данные из многих       → низкий шум
  
В CSV файле count_scipy.csv видно эту информацию!
```

## Критическая операция: Почему NaN на краях правильно?

```
СЦЕНАРИЙ: Интерполировать V вне диапазона периода?

ВАРИАНТ А: Экстраполяция (ПЛОХО ❌)
  Период имеет H ∈ [180, 200]
  Линейное продолжение:
    H=210: V = 20000 + (20000-19000)/(200-180) × (210-200) = 21000 л
    H=220: V = 22000 л
    ...
    H=300: V = 31000 л
  
  ⚠️ ПРОБЛЕМА: Реальный объём при H=300 ~ 31416 л (для R=100)
  но это совпадение! Для других R экстраполяция даст полный вздор!

ВАРИАНТ B: NaN (ХОРОШО ✓)
  Период имеет H ∈ [180, 200]
  Вне диапазона → NaN
  
  ✓ Честно: "Нет данных вне [180, 200]"
  ✓ В Этапе 6 (агрегация) эти узлы просто пропускаются
  ✓ Если другие периоды покрывают эту высоту → будут данные
  ✓ Если никто не покрывает → узел останется пустым в combined_characteristics

РЕЗУЛЬТАТ:
  combined_characteristics имеет узлы ТОЛЬКО там, где есть хотя бы 1 период
  Это — самое честное и безопасное решение!
```

## Итоговая диаграмма: От данных к результату

```
ВХОД: 8 периодов по 20 точек = 160 точек (H,V) с шумом
     + параметры (grid_step=1, 3 алгоритма, seed=42)

        ⬇ ОБРАБОТКА

ЭТАП 4: Развёртывание + сглаживание
  160 → 160 (одна форма, более гладкая)
```
