# Быстрая шпаргалка: BenzoSim за 2 минуты

## Что это?

**BenzoSim** восстанавливает характеристику V(H) (объём по уровню) из нескольких независимых периодов опустошения ёмкости.

## Как запустить?

```bash
python main_periodic.py
```

Результаты → `plots/periodic_*/`

## 9 этапов пайплайна

| Этап | Что делает | Ключевая идея |
|------|-----------|---------------|
| **1** | Инициализация | Подготавливаем геометрию, конфиг, RNG |
| **2** | Генерация периодов | Генерируем 5-8 случайных "сливов" ёмкости + шум |
| **3** | Статистика | Выводим информацию о периодах |
| **4** | Сглаживание | Сглаживаем данные КАЖДОГО периода тремя алгоритмами |
| **5** | ⭐ **Интерполяция** | Интерполируем периоды на единую сетку (1см шаг) |
| **6** | ⭐ **Агрегация** | Объединяем ВСЕ периоды → одна V(H) |
| **7** | Сравнение | Сравниваем с эталонной геометрией |
| **8** | Производные | Анализируем dV/dH |
| **9** | Визуализация | PNG графики + CSV файлы |

## ⭐ Главное: Как интерполируются границы?

### Проблема в одной строке:
Периоды — это **убывающие** последовательности (опустошение идёт сверху вниз), но алгоритмы требуют **возрастающих**!

### Решение:

```python
# Этап 4: Сглаживание
H_measured = [200, 195, 190, ...]  # убывает
H_ascending = H_measured[::-1]     # [190, 195, 200] ← РАЗВЕРНУЛИ!
# Сглаживаем, потом развернули обратно

# Этап 5: Интерполяция на сетку
# Период покрыл [180, 200]
H_grid = [0, 1, 2, ..., 179, 180, 181, ..., 200, 201, ...]
V_grid = [NaN, NaN, ..., NaN, 18000, 18100, ..., 20000, NaN, ...]
         └─ нет данных ──┬──── интерполяция ────┬────── нет данных ──┘
                      [180,200]              вне диапазона

# Ключ: интерполируем ТОЛЬКО внутри диапазона!
```

### Почему NaN за пределами?

- **Внутри [180, 200]** → есть реальные измерения → линейная интерполяция надёжна
- **Вне диапазона** → нет данных → экстраполяция опасна → NaN (честно говорим "нет")

## Агрегация (Этап 6): Как объединить периоды?

```
5 периодов, узел H=100:

Облако точек:
  Период 1: V = 12000
  Период 2: V = 12050  
  Период 3: V = 11950  ← выброс
  Период 4: V = 12100
  Период 5: V = 12000

Усреднение: V_avg = (12000+12050+11950+12100+12000)/5 = 12020
                     └─ выброс тянет вниз, но средн остаётся близко к истине
```

## Три алгоритма сглаживания

| Алгоритм | Метод | Скорость | Гладкость | Когда использовать |
|----------|-------|----------|-----------|-------------------|
| **scipy** | UnivariateSpline | ⚡ Быстро | 🔵🔵🔵 Нормально | Когда спешим |
| **original_sg1** | Конечные разности | 🐌 Медленно | 🔵🔵 Менее гладко | Диагностика |
| **original_sg2** | M-сплайны | 🐌 Медленно | 🔵🔵🔵🔵 Гладко | Когда нужна гладкость |

## Параметры конфига (главные)

```python
n_periods = 8           # Больше = надёжнее, но дольше
grid_step = 1.0         # Шаг сетки [см], 0.5 → точнее но шумнее
level_precision = 1     # Округление уровней, 0 → нет округления

# Параметры "опустошения"
period_start_min/max = 80-180 см    # Где начинаем слив
period_end_min/max = 20-60 см       # Где заканчиваем слив
min/max_drain_step = 0.06-3.0 см    # Размер шагов опустошения

# Шумы (реалистичные значения)
Del_h = 0.05 см        # Шум по высоте
Del_V = 0.0015         # Относительный шум объёма (0.15%)
```

## Как проверить, что работает?

1. **Откройте** `combined_vs_geometry_scipy.png`
   - Красная кривая (восстановленная) должна быть **близко** к синей (точной)
   - Если далеко → проверьте n_periods, grid_step

2. **Откройте** `counts_scipy.csv`
   - Большие значения (count > 3) → надёжные узлы
   - count = 1 → может быть шумным

3. **Откройте** `cloud_before_agg_scipy.csv`
   - Для одного узла должно быть несколько V из разных периодов
   - Если они сильно разнятся → шум, но усреднение поможет

## Что означают CSV файлы?

### `combined_scipy.csv` (основной результат)

```

0.0,0.0

...
150.0,14950     ← При уровне 150см объём ≈ 14950л

```

### `counts_scipy.csv` (надёжность)

```

151.0,4       ← 4 периода (ещё хорошо)
152.0,2       ← 2 периода (менее надёжно)
```

### `cloud_before_agg_scipy.csv` (сырые точки)

```

150.0,14950,1
150.0,15000,2
150.0,14900,5
        ↑
      Три измерения для одной высоты из разных периодов
      Среднее = (14950+15000+14900)/3 ≈ 14950
```

## Диагностика проблем

| Проблема | Причина | Решение |
|----------|---------|---------|
| `combined_vs_geometry.png` не совпадают | Мало периодов или мелкая сетка | ↑ n_periods, ↓ grid_step |
| Много "дыр" в `combined.csv` | Периоды не покрывают весь диапазон | ↑ n_periods или ↑ период_start_max, ↓ period_end_min |
| Кривая очень шумная | Мало периодов или большой шум | ↑ n_periods или применить усиленное сглаживание |
| Код работает медленно | Слишком много periодов или алгоритмов | ↓ n_periods, использовать только scipy |

## Помню ещё с прошлой итерации?

Вот твой главный вопрос:

> **"Каким образом мы интерполируем границы периодов в заданную "ровную" сетку?"**

**Ответ в 3 пункта:**

1. **Развёртываем** убывающие последовательности в возрастающие
2. **Сглаживаем** и удаляем дубликаты
3. **Интерполируем на сетку ТОЛЬКО ВНУТРИ диапазона периода**, вне → NaN
4. **Разворачиваем обратно** для согласованности
5. Потом агрегируем ВСЕ периоды через **облако точек и усреднение**

**Ключ:** Интерполяция честна — показывает через NaN, где нет данных!
